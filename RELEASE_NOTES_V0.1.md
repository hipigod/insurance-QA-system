## 🎉 版本概述

这是保险销售智能陪练系统的首个正式版本,提供完整的AI驱动销售训练功能。

**核心价值**: 通过AI模拟真实客户场景,帮助保险销售人员提升销售技能。

---

## ✨ 主要功能

### 1. 多样化客户角色 ✅

提供4种不同难度的客户角色,从简单到困难循序渐进:

- **小白客户** (简单)
  - 对保险了解不多
  - 需要耐心解释
  - 适合新手练习

- **理性分析型** (中等)
  - 注重数据和细节
  - 需要专业说明
  - 考验产品知识

- **价格敏感型** (困难)
  - 预算有限
  - 需要价值塑造
  - 考验沟通技巧

- **怀疑抗拒型** (困难)
  - 对行业有偏见
  - 需要建立信任
  - 考验异议处理

### 2. 完整的保险产品库 ✅

提供3种典型保险产品用于练习:

- **少儿教育金保险**
  - 教育储蓄 + 保障
  - 保费: 5000-20000元/年
  - 目标: 0-12岁儿童父母

- **终身重大疾病保险**
  - 100+种重疾保障
  - 保费: 3000-15000元/年
  - 目标: 18-55岁健康人群

- **定期寿险**
  - 低保费高保障
  - 保费: 500-3000元/年
  - 目标: 20-50岁家庭支柱

### 3. 多维度评分体系 ✅

4个评分维度全面评估销售能力:

1. **需求挖掘** (20%)
   - 提问质量
   - 倾听能力
   - 需求确认
   - 针对性推荐

2. **产品说明** (25%)
   - 准确性
   - 清晰度
   - 针对性
   - 证据支持

3. **异议处理** (25%)
   - 情绪管理
   - 异议识别
   - 处理技巧
   - 效果评估

4. **促成能力** (30%)
   - 时机把握
   - 促成技巧
   - 临门一脚
   - 专业性

### 4. AI智能对话 ✅

- **AI引擎**: 通义千问 qwen-plus
- **实时对话**: WebSocket双向通信
- **角色扮演**: AI模拟真实客户反应
- **智能评分**: 实时分析对话质量

### 5. 完整管理后台 ✅

- **角色管理**: 创建、编辑、删除客户角色
- **产品管理**: 管理保险产品信息
- **维度管理**: 调整评分维度和权重
- **案例库**: 管理优秀销售案例
- **数据导出**: 支持JSON和CSV格式

### 6. 模型配置管理 ✅

- **多模型支持**: 可配置不同AI模型
- **API管理**: 管理API Key和地址
- **模型切换**: 一键激活不同模型
- **安全存储**: API Key加密存储

## 🔧 技术栈

### 前端
- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite 5.4.21
- **UI库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **实时通信**: WebSocket API

### 后端
- **框架**: FastAPI 0.109.0
- **服务器**: Uvicorn 0.27.0
- **ORM**: SQLAlchemy 2.0.45 (异步)
- **数据库**: SQLite (aiosqlite)
- **数据验证**: Pydantic 2.12.5
- **AI集成**: OpenAI SDK (兼容通义千问)

### AI模型
- **主模型**: 通义千问 qwen-plus
- **API地址**: https://dashscope.aliyuncs.com/compatible-mode/v1
- **备选模型**: 支持OpenAI、DeepSeek、Claude等

## 📦 安装部署

### 系统要求
- Python 3.13+
- Node.js 18+
- 操作系统: Windows/Mac/Linux

### 快速开始

1. **克隆项目**
   ```bash
   git clone <repository-url>
   cd Insurance Q&A System
   ```

2. **安装依赖**
   ```bash
   # 后端依赖
   cd backend
   pip install -r requirements.txt

   # 前端依赖
   cd ../frontend
   npm install
   ```

3. **初始化数据**
   ```bash
   cd ..
   python add_preset_data.py
   python add_model_config.py
   ```

4. **启动服务**
   ```bash
   # 后端 (终端1)
   cd backend
   python main.py

   # 前端 (终端2)
   cd frontend
   npm run dev
   ```

5. **访问系统**
   - 前端: http://localhost:5173
   - API文档: http://127.0.0.1:8000/docs
   - 管理后台: http://localhost:5173/#/admin

详细说明请参考 [QUICKSTART.md](QUICKSTART.md)

## 🐛 已知问题

1. **Windows控制台编码问题**
   - 问题: emoji字符在某些Windows终端显示为乱码
   - 影响: 仅影响控制台显示,不影响功能
   - 解决: 使用Git Bash或PowerShell,或忽略显示警告

2. **模型配置API需要重启**
   - 问题: 新增的模型配置API需要重启后端才能生效
   - 影响: 首次使用时需要重启
   - 解决: 重启后端服务即可
